## 문제 풀이

주어진 정수 `n`에 대해 제곱했을 때 `n`보다 크거나 같은 음이 아닌 정수 `q`를 찾는 문제다. 단순히 `Math.sqrt()`를 활용하기에는 정수 `n`의 크기가 너무 커서 소수점 정밀도 문제가 있다. 해당 문제는 이분 탐색과 매개 변수 탐색을 활용해서 해결했는데 `q`의 하한과 상한을 정하고 제곱한 값에 대해 `n`보다 크거나 같아지는 가장 작은 수가 될 때까지 탐색했다.

`q`의 하한은 `0`으로 잡고 상한은 `(long) Math.ceil(Math.sqrt(Long.MAX_VALUE))`로 잡았는데 정수 `n`이 `long` 타입의 최댓값보다는 작으니 `long` 타입의 최댓값의 제곱근을 올림한 값보다는 작을 것으로 생각했다.

---

## 코드

```java
import java.io.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        long n = Long.parseLong(br.readLine());

        System.out.println(lowerBound(n));
    }

    private static long lowerBound(long key) {
        long left = 0;
        long right = (long) Math.ceil(Math.sqrt(Long.MAX_VALUE));

        while (left < right) {
            long mid = (left + right) / 2;

            if (mid * mid < key) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }

        return right;
    }
}
```

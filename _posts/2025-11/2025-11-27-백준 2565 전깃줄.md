## 문제 풀이

교차하지 않도록 제거해야 하는 전깃줄의 최소 개수를 구해야 하는 문제로 전깃줄이 교차하지 않을 경우 `A`의 위치 번호가 증가할수록 연결된 `B`의 위치 번호도 증가한다는 점에서 LIS를 활용하면 해결할 수 있다. 전체 전깃줄의 개수에서 LIS를 이루는 전깃줄의 개수를 빼면 없애야하는 전깃줄의 개수를 구할 수 있다.

---

## 코드

### 1. 2중 for문 LIS [Java]

2차원 배열로 전깃줄을 받아서 `A`의 위치를 기준으로 오름차순으로 정렬해줬다. 이러면 `B`의 위치에 대한 LIS가 교차하지 않는 전깃줄의 최대 개수가 된다.

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st;

        int N = Integer.parseInt(br.readLine());

        int[][] map = new int[N][2];
        for (int i = 0; i < N; i++) {
            st = new StringTokenizer(br.readLine());
            map[i][0] = Integer.parseInt(st.nextToken());
            map[i][1] = Integer.parseInt(st.nextToken());
        }
        Arrays.sort(map, (o1, o2) -> Integer.compare(o1[0], o2[0]));

        int[] dp = new int[N];
        Arrays.fill(dp, 1);
        int max = 0;

        for (int i = 0; i < N; i++) {
            for (int j = 0; j < i; j++) {
                if (map[i][1] > map[j][1]) dp[i] = Math.max(dp[i], dp[j] + 1);
            }

            max = Math.max(max, dp[i]);
        }

        System.out.println(N - max);
    }
}
```

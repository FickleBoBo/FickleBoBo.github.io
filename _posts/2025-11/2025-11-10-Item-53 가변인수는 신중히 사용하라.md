## 가변인수(varargs) 메서드

가변인수 메서드는 명시한 타입의 인수를 0개 이상 받을 수 있는 메서드다. 가변인수 메서드를 호출하면, 가장 먼저 인수의 개수와 길이가 같은 배열을 만들고 인수들을 이 배열에 저장하여 가변인수 메서드에 건네준다.

아래는 가변인수 메서드를 활용하여 파라미터로 넘어온 변수들의 합을 출력하는 예시로 다양한 개수의 파라미터에도 하나의 메서드로 유연하게 처리할 수 있다.

```java
public class VarargsMain1 {
    public static void main(String[] args) {
        int result1 = sum();
        int result2 = sum(1);
        int result3 = sum(1, 2, 3);

        System.out.println("result1 = " + result1);  // result1 = 0
        System.out.println("result2 = " + result2);  // result2 = 1
        System.out.println("result3 = " + result3);  // result3 = 6
    }

    static int sum(int... args) {
        int sum = 0;
        for (int arg : args) {
            sum += arg;
        }
        return sum;
    }
}
```

`List`의 정적 팩터리 메서드인 `of`도 가변인수 메서드로 다양한 개수의 파라미터를 받을 수 있다. 오버로딩된 메서드가 많은 것은 가변인수 메서드를 통해 배열을 만들고 저장하는 과정도 최적화한 것이다. 대부분의 리스트 생성에서 파라미터의 수가 10개 이하라면 가변인수 메서드의 성능 문제를 회피할 수 있다.

![](/assets/posts/2025-11/Item-53%20가변인수는%20신중히%20사용하라/assets/photo1.png)

또 자주 사용하는 `System.out.printf` 메서드 역시 가변인수 메서드로 이 덕분에 다양한 값들을 원하는 양식으로 출력할 수 있다.

![](/assets/posts/2025-11/Item-53%20가변인수는%20신중히%20사용하라/assets/photo2.png)

사실 `main` 메서드로 가변인수 메서드다. 아래처럼 파라미터를 가변인수로 설정하고

```java
public class VarargsMain0 {
    public static void main(String... args) {
        int sum = 0;
        for (String arg : args) {
            sum += Integer.parseInt(arg);
        }
        System.out.println(sum);
    }
}
```

파라미터를 넘겨주면 잘 출력되는 것을 볼 수 있다.

```bash
 java VarargsMain0 1 2 3 4 5
15
```
